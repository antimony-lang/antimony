Then add a new test function `test_qbe_execution_tests()` (per D-03):
```rust
#[test]
fn test_qbe_execution_tests() -> Result<(), Error> {
    let dir = std::env::current_dir().unwrap();
    let dir_out = dir.join("tests_qbe_out");
    let _ = fs::create_dir(&dir_out);

    let test_dir = dir.join("tests/qbe");
    let tests = std::fs::read_dir(&test_dir)?;

    for entry in tests {
        let entry = entry?;
        let path = entry.path();
        if path.is_dir() || path.extension().map_or(true, |e| e != "sb") {
            continue;
        }
        compile_and_run_qbe_checked(&path, &dir_out)?;
    }
    Ok(())
}
```

- `<`: 1 < 2 is true, 2 < 1 is false, 1 < 1 is false

- `!=`: 5 != 6 is true, 5 != 5 is false

**test_booleans.sb** -- Tests Bool type, `&&`, `||` operators:
- `let t = true` and `let f = false` -- basic bool assignment
- `true && true` is true, `true && false` is false, `false && false` is false
- `true || false` is true, `false || false` is false
- Nested: `(true && true) || false` is true

**test_variables.sb** -- Tests Declare and Assign statements:
- `let x = 42` then use x in comparison
- `let y: int = 10` (typed declaration)
- Reassignment: `x = 100` then verify x == 100
- Multiple variables interacting: `let a = 3` `let b = 4` `let c = a + b` verify c == 7

**test_if_else.sb** -- Tests If statement with optional else:
- Simple if (true condition) -- body executes
- Simple if (false condition) -- body does not execute
- If-else: verify correct branch taken
- Nested if-else chains (if/else if/else)

**test_while_loops.sb** -- Tests While statement, Break, Continue:
- Basic while loop counting to 5
- While loop with break (exit at 3)
- While loop with continue (skip even numbers, count odds)
- Nested while loops

**test_functions.sb** -- Tests Function definitions, FunctionCall, Return:
- Function with no args returning int
- Function with args: `fn add(a: int, b: int): int { return a + b }`
- Recursive function (factorial or fibonacci, small values)
- Void function (side effect via variable mutation -- but Antimony uses value semantics, so test void functions that just compute and return nothing, verify they don't crash)

**test_strings.sb** -- Tests Str type and string builtins:
- String literal assignment: `let s = "hello"`
- String concatenation via `+` operator (or str_concat builtin if + doesn't work for strings)
- `strlen()` or equivalent
- `int_to_str()` conversion
- Print string with println
