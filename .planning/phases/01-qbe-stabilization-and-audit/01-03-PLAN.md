From qbe crate (current Type enum):
```rust
pub enum Type<'a> {
    Word, Long, Single, Double, Zero,
    Byte, SignedByte, UnsignedByte,
    Halfword, SignedHalfword, UnsignedHalfword,
    Aggregate(&'a TypeDef<'a>),
}
```

The fix needs to add an owned variant (e.g., OwnedAggregate(Rc<TypeDef<'static>>)) so callers don't need to transmute lifetimes.

  <action>
This task requires the user to fix and publish their `qbe` crate. Claude cannot publish crates.

The qbe crate needs a fix to support owned aggregate types, eliminating the need for `std::mem::transmute` when creating `Type::Aggregate` values from `Rc&lt;TypeDef&gt;`.

**What to change in the qbe crate:**

Option A (recommended -- add owned variant):
Add to `pub enum Type&lt;'a&gt;` in the qbe crate's `src/lib.rs`:
```rust
/// Owned aggregate type backed by Rc, no lifetime borrowing needed
OwnedAggregate(std::rc::Rc&lt;TypeDef&lt;'static&gt;&gt;),
```
Update all match arms on `Type` to handle `OwnedAggregate` the same as `Aggregate` (emit the same QBE IL text). The `Display` impl for `Type` should format `OwnedAggregate` identically to `Aggregate`.

Option B (alternative -- change Aggregate to owned):
Change `Aggregate(&amp;'a TypeDef&lt;'a&gt;)` to `Aggregate(std::rc::Rc&lt;TypeDef&lt;'static&gt;&gt;)` and remove the lifetime parameter from `Type` entirely. This is a breaking change but simpler.

**After making the change:**
1. Bump the version in the qbe crate's Cargo.toml (e.g., 3.0.0 -&gt; 3.1.0 for Option A, or 4.0.0 for Option B)
2. `cargo publish` the updated crate
3. Note the new version number
