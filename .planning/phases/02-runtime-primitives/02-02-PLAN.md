Append to `builtin/builtin_qbe.c` after the `_read_line` function:

```c
/* _str_char_at(s, idx) — return the character at index idx as a 1-char string */
char *_str_char_at(char *s, long idx)
{
    char *buf = malloc(2);
    buf[0] = s[idx];
    buf[1] = '\0';
    return buf;
}

/* _str_substr(s, start, len) — extract a substring [start, start+len) */
char *_str_substr(char *s, long start, long len)
{
    char *buf = malloc(len + 1);
    memcpy(buf, s + start, len);
    buf[len] = '\0';
    return buf;
}
```

**Part A: Add _malloc QBE IL preamble wrapper (per D-02)**

In `src/generator/qbe.rs`, append to `RUNTIME_PREAMBLE`:

```
# _malloc(size: w): l — allocate size bytes on the heap, return pointer as long
export function l $_malloc(w %size) {
@start
    %sz =l extsw %size
    %ptr =l call $malloc(l %sz)
    ret %ptr
}
```

Note: Takes `w` (word/int) for size since Antimony's int is word-sized, but extends to `l` for the libc malloc call. Returns `l` (long/pointer).

**Part B: Add return type to infer_builtin**

In `src/parser/infer.rs` `infer_builtin`, add:
- `"_malloc"` => `Some(Type::Str)` — returns pointer-as-long. The string type maps to `qbe::Type::Long` (64-bit), which is required to hold the 64-bit heap pointer without truncation. Using `Type::Int` would truncate to 32-bit `qbe::Type::Word`, corrupting the pointer on 64-bit systems.

**Part C: Create test_malloc.sb**

Create `tests/qbe/test_malloc.sb` — a self-checking test that:
1. Calls `let ptr: string = _malloc(64)` to allocate 64 bytes (string type holds 64-bit pointer)
2. Verifies ptr is not 0 (non-null check: `if ptr == 0 { ... FAIL }`)
3. Allocates a second block: `let ptr2: string = _malloc(128)`
4. Verifies ptr2 is not 0 and ptr2 != ptr (different allocations)
5. Prints "PASS" and exits 0 if all correct, "FAIL" and exits 1 on failure

Note: This test cannot write to or read from the allocated memory without pointer-dereference support, which Antimony doesn't have yet. The test validates that malloc returns non-null distinct pointers, which proves the allocation mechanism works. Phase 3 stdlib (dynamic arrays) will exercise actual memory use through C helper functions.

IMPORTANT: The `string` type is used as the opaque pointer type because Antimony's `string` maps to `qbe::Type::Long` (64-bit), which correctly holds a 64-bit heap address. Using `int` (which maps to `qbe::Type::Word`, 32-bit) would silently truncate the pointer.
